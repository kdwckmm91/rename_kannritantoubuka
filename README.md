# 台帳管理担当部課一括変換ツール

利用者名または組織管理区分１名をキーとして、台帳ファイルの「管理担当部課」と「管理担当部課名」を一括変換するツールです。

## 概要

このツールは2つの実装方法を提供します：

1. **Python スクリプト版** (`main.py`) - 推奨
   - 対応関係ファイルから自動的に読み込んで変換
   - 変更履歴がログ出力される
   - バッチ処理に向いている

2. **Excel マクロ版** (`macro_template.vba`)
   - Excel 内でマクロを実行
   - Excel を直接操作できる

## ファイル構成

```
sisantantou/
├── main.py                          # Pythonスクリプト（メイン）
├── mapping_master.xlsx              # 対応関係ファイル（ひな形）
├── macro_template.vba               # VBA マクロコード
├── マクロ用テンプレート.xlsx        # VBA 埋め込み用テンプレート
├── 台帳_20260616.xlsx               # 元の台帳ファイル（入力）
├── 台帳_20260616_updated.xlsx       # 更新後の台帳ファイル（出力）
└── README.md                        # このファイル
```

## 使用方法

### Method 1: Python スクリプト版（推奨）

#### ステップ1: 対応関係ファイルの作成

`mapping_master.xlsx` ファイルを開いて、対応関係データを入力します。

| キー | 新コード | 新名称 |
|---|---|---|
| 利用者A | 822108020001 | 営業部 |
| 利用者B | 822108020002 | 企画部 |
| 組織管理区分X | 822108020003 | 技術部 |

**入力ルール:**

- 「キー」列: 台帳の「利用者名」または「組織管理区分１名」と完全一致する値
- 「新コード」列: 数値（例：822108020001）
- 「新名称」列: テキスト（例：営業部）

#### ステップ2: Python スクリプトを実行

コマンドプロンプト/ターミナルで：

``` bash
cd c:\Temp\ツール\sisantantou
python main.py
```

または、仮想環境を使用している場合：

``` bash
cd c:\Temp\ツール\sisantantou
.venv\Scripts\python.exe main.py
```

**実行結果:**

- `台帳_20260616_updated.xlsx` が生成されます
- コンソールにログが出力されます（更新行数など）

#### ステップ3: 結果確認

`台帳_20260616_updated.xlsx` を開いて、以下の列が更新されていることを確認します：

- **C列**: 管理担当部課（新しい部課コードに更新）
- **D列**: 管理担当部課名（新しい部課名に更新）

### Method 2: Excel マクロ版

#### ステップ1: VBA マクロを埋め込み

1. `マクロ用テンプレート.xlsx` を開く
2. **Alt + F11** でマクロエディタを開く
3. **Project > Insert > Module** で新しいモジュールを追加
4. `macro_template.vba` のコード全体をコピーしてペースト
5. **Ctrl + S** でファイルを保存

#### ステップ2: マッピングシートにデータを入力

1. `マクロ用テンプレート.xlsx` の「マッピング」シートを開く
2. 対応関係データを入力（上記「Python スクリプト版」の表を参照）
3. ファイルを保存

#### ステップ3: マクロを実行

1. Excel で **Alt + F8** を押すか、**Developer > Macros** から実行
2. `UpdateLedgerFromMapping` マクロを選択して実行
3. メッセージボックスに完了結果が表示されます

#### ステップ4: 結果確認

メインシート（シート1）の C列・D列が更新されていることを確認します。

## 対応関係ファイルの形式

### 入力例

利用者名をキーとした場合：

- キー: 佐藤太郎
- 新コード: 822108020001
- 新名称: 営業部

組織管理区分１名をキーとした場合：

- キー: 営業推進部
- 新コード: 822108020002
- 新名称: 営業推進部

### 処理フロー

```
対応関係ファイル → マッチング処理
    ↓
1. 台帳の「利用者名」と対応関係の「キー」を比較
2. マッチした場合、C列・D列を更新
3. マッチしない場合、「組織管理区分１名」で再度比較
4. マッチした場合、C列・D列を更新
```

## トラブルシューティング

### Python スクリプトが実行できない

**症状:** `python: command not found` または権限エラー

**解決方法:**

``` bash
# 絶対パスで実行
c:\Temp\ツール\.venv\Scripts\python.exe main.py
```

### 対応関係ファイルが読み込めない

**症状:** `対応関係ファイルが見つかりません`

**確認項目:**

- `mapping_master.xlsx` が sisantantou ディレクトリに存在するか
- ファイル名が正確に入力されているか
- Excel で開いている場合は、一度閉じてから実行

### 更新されない行がある

**原因:** 対応関係ファイルの「キー」と台帳のデータが完全一致していない

**確認方法:**

- 対応関係ファイルの「キー」をコピー
- 台帳の「利用者名」または「組織管理区分１名」と比較
- 空白文字や大文字小文字の違いがないか確認

### マクロが実行されない

**症状:** マクロが見つからない、または「セキュリティ警告」が表示

**解決方法:**

1. Excel を終了
2. ファイルを右クリック → プロパティ
3. 「ブロック解除」をチェック → OK
4. 再度 Excel で開く
5. マクロセキュリティ設定で「マクロを有効にする」を選択

## 注意事項

1. **バックアップ**: 元の台帳ファイルは `台帳_20260616.xlsx` のまま保持されます
2. **データ型**: 部課コードは数値型で保存されます（Excel の数値フォーマットに注意）
3. **大文字小文字**: キーの比較は大文字小文字を区別しません
4. **空白文字**: キーの前後の空白は自動的に削除されます
5. **二重実行**: 同じ対応関係で複数回実行すると、同じ結果が上書きされます

## 出力ファイル

Python スクリプト実行後、以下のファイルが生成されます：

- **台帳_20260616_updated.xlsx**: 更新後の台帳
- コンソールログに以下の情報が出力:
  - 更新行数（合計）
  - 利用者名で突合した行数
  - 組織管理区分１名で突合した行数
  - 非突合行数

## 仕様

### 対応関係ファイル構造

| 列 | 名称 | データ型 | 例 |
|---|---|---|---|
| A | キー | 文字列 | 利用者A / 営業部 |
| B | 新コード | 数値 | 822108020001 |
| C | 新名称 | 文字列 | 営業部 |

### 台帳ファイル（変更対象列）

| 列 | 名称 | 変更内容 |
|---|---|---|
| C | 管理担当部課 | 対応関係ファイルの「新コード」で更新 |
| D | 管理担当部課名 | 対応関係ファイルの「新名称」で更新 |

### キー列（突合対象）

| 列 | 名称 | 優先度 |
|---|---|---|
| 利用者名（69列目） | 第1優先 |
| 組織管理区分１名（82列目） | 第2優先 |
